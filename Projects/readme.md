# 📁 Carpeta de proyectos

## 📖 Descripción  
Esta carpeta agrupa los proyectos principales desarrollados como parte del curso de **Node.js** de **SergieCode**. Cada proyecto representa una aplicación práctica de los conocimientos adquiridos durante el curso, permitiendo consolidar conceptos relacionados con el desarrollo backend, la creación de servidores, el diseño de APIs REST, la gestión de bases de datos y la autenticación de usuarios mediante tecnologías modernas del ecosistema de Node.js.

Los proyectos aquí almacenados abarcan desde la simulación de una API mediante un archivo JSON, hasta el desarrollo de servidores con **Express**, la creación de APIs REST conectadas a **MongoDB** mediante **Mongoose**, y finalmente un sistema completo de autenticación y gestión de usuarios utilizando **TypeScript**, **Prisma**, **PostgreSQL** y **JSON Web Tokens (JWT)**. Cada uno fue diseñado para reforzar diferentes aspectos del desarrollo backend y comprender cómo se construyen aplicaciones del mundo real.

---

## 📂 Estructura  

- **`Project_1/`**:  
  Proyecto de **API simulada con JSON Server**, donde se crea una base de datos utilizando un archivo `db.json` para administrar información sobre profesores, estudiantes y cursos. Este proyecto permite comprender el funcionamiento de APIs REST simuladas, el manejo de recursos mediante peticiones HTTP y la organización de datos estructurados para realizar operaciones CRUD durante el desarrollo y las pruebas.

- **`Project_2/`**:  
  Proyecto de **servidor web con Express**, donde se construye un servidor capaz de servir archivos estáticos utilizando **Express**, gestionar variables de entorno mediante **dotenv** y **env-var**, configurar rutas por defecto y estructurar el proyecto utilizando módulos independientes. Este proyecto fortalece la comprensión del funcionamiento interno de Express, la configuración de servidores y la organización profesional de aplicaciones backend.

- **`Project_3/`**:  
  Proyecto de **API REST para gestión de libros**, desarrollada con **Express**, **MongoDB**, **Mongoose** y **Docker**. La aplicación implementa operaciones CRUD completas para administrar una colección de libros, utilizando modelos, rutas, middlewares personalizados, validación de identificadores y conexión a una base de datos NoSQL. Además, se emplea Docker para facilitar el despliegue del entorno de desarrollo y la administración de la base de datos.

- **`Project_4/`**:  
  Proyecto de **API REST con autenticación y gestión de usuarios**, desarrollado con **TypeScript**, **Express**, **Prisma ORM**, **PostgreSQL**, **JWT** y **bcrypt**. La aplicación implementa un sistema completo de registro e inicio de sesión con contraseñas cifradas, autenticación basada en tokens, protección de rutas mediante middleware, operaciones CRUD sobre usuarios y acceso seguro a la base de datos utilizando Prisma. Este proyecto integra múltiples tecnologías modernas para simular una arquitectura backend utilizada en aplicaciones profesionales.

---

## 🔧 Tecnologías y Herramientas  
- Node.js como entorno de ejecución para JavaScript del lado del servidor.  
- Express.js para la creación de servidores web y APIs REST.  
- TypeScript para añadir tipado estático y mejorar la mantenibilidad del código.  
- MongoDB junto con Mongoose para el modelado y gestión de bases de datos NoSQL.  
- PostgreSQL junto con Prisma ORM para el acceso y administración de bases de datos relacionales.  
- Docker para la ejecución de servicios y bases de datos en contenedores.  
- JSON Server para la simulación de APIs REST durante el desarrollo.  
- Dotenv y Env-var para la gestión segura de variables de entorno.  
- JSON Web Tokens (JWT) y bcrypt para la autenticación y protección de usuarios.
